**Проект:** FlashCard Studio (Редактор карточек для запоминания)
**Платформа:** Десктоп (Windows / macOS / Linux)
**Стек:** C++, Qt 6 (QtCore, QtGui, QtWidgets). Строго без JavaScript.

---

## 1. Архитектура данных и хранение (Формат `.flcr`)
Формат файла `.flcr` представляет собой текстовый файл, содержащий данные в формате **JSON** (формируется через `QJsonDocument` и `QJsonObject`).

### 1.1. Структура JSON:
*   **ProjectName** (string): Название проекта.
*   **LastModified** (string/ISO-8601): Дата последнего изменения.
*   **Decks** (array): Массив колод (плоский список).
    *   **DeckID** (string/UUID): Уникальный ID.
    *   **Name** (string): Имя колоды.
    *   **Cards** (array): Массив карточек.
        *   **CardID** (string/UUID).
        *   **Front** (string/Rich Text): Лицевая сторона (содержит HTML-теги форматирования от `QTextEdit`).
        *   **Back** (string/Rich Text): Обратная сторона.
        *   *Метаданные для алгоритма обучения (SM-2):*
            *   **Repetitions** (int): Количество успешных повторений подряд.
            *   **EaseFactor** (float): Коэффициент легкости (по умолчанию 2.5).
            *   **Interval** (int): Текущий интервал до следующего показа в днях.
            *   **NextReviewDate** (string/ISO-8601): Дата следующего показа (Epoch time или строка).

## 2. Глобальный дизайн и Иконки
Чтобы не перегружать интерфейс текстом, приложение должно использовать минималистичный набор иконок (например, Google Material Icons или FontAwesome, загруженные через файл `resources.qrc`).
*   Цветовая палитра: Светлая тема (или системная), акцентный цвет (например, синий, как в MS Word / VS Code).
*   Вместо текстовых кнопок "Добавить", "Удалить", "Настройки" везде используются иконки с всплывающими подсказками (`setToolTip()`).

## 3. Описание экранов и UI (Qt Widgets)

### 3.1. Стартовый экран (Backstage View)
*Открывается при запуске без файла. Занимает все окно.*
*   **Левая панель (Акцентный цвет):**
    *   Иконка приложения + Название.
    *   Большая кнопка: `[Иконка 📄] Создать проект` (Вызывает `QFileDialog::getSaveFileName`).
    *   Большая кнопка: `[Иконка 📂] Открыть проект` (Вызывает `QFileDialog::getOpenFileName`).
*   **Правая панель (Белый/Светлый фон):**
    *   Заголовок: "Недавние проекты" (с иконкой 🕒).
    *   Список `QListWidget`: Содержит элементы с иконкой `.flcr` файла, именем проекта и путем. Клик по элементу открывает проект и переключает на Главное окно.

### 3.2. Главное окно (Main Window)
*Класс: `QMainWindow`. Открывается после загрузки проекта.*
*   **Классическое меню (`QMenuBar`):**
    *   **Файл:** Создать, Открыть, Сохранить (`Ctrl+S`), Сохранить как, На стартовый экран, Выход. (Ко всем пунктам привязаны иконки).
    *   **Правка:** Отменить, Повторить.
    *   **Справка:** О программе.
*   **Левая боковая панель (`QDockWidget` - Список колод):**
    *   **Верхняя часть панели:** Поле `QLineEdit` (placeholder: "🔍 Поиск колоды...") и кнопка `QToolButton` `[Иконка ➕]` (Добавить колоду).
    *   **Список (`QListWidget`):** Плоский список колод. 
    *   **Поведение элемента списка:** 
        *   Отображает иконку 🗂️ и название колоды.
        *   При наведении курсора (`hover`) справа появляются две кнопки (реализуется через кастомный `QStyledItemDelegate` или `setItemWidget` с кастомным `QWidget`):
            1.  `[Иконка ✏️]` - Открыть в редакторе (ToolTip: Редактировать).
            2.  `[Иконка ▶️]` - Открыть в обучении (ToolTip: Учить).
*   **Центральная область (`QStackedWidget`):**
    *   Имеет 3 страницы: 1. Пустая заглушка (иконка логотипа по центру), 2. Экран редактирования, 3. Экран обучения.

### 3.3. Режим редактирования (Edit Mode)
*Открывается в `QStackedWidget` при нажатии на ✏️.*
*   **Верхний ToolBar (`QToolBar` внутри виджета):**
    *   Название колоды (крупный шрифт, `QLabel`).
    *   `[Иконка ➕]` Добавить карточку (Ctrl+N).
    *   `[Иконка 🗑️]` Удалить выбранную карточку (Del).
    *   Распорка (`QSpacerItem`), затем `QLabel` со статистикой: "Всего карточек: N".
*   **Список карточек (`QTableView` + `QAbstractTableModel`):**
    *   Занимает верхнюю половину области.
    *   Колонки: `ID` (скрыта), `Вопрос` (превью текста без HTML), `Ответ` (превью).
    *   Поведение: Выделяется только целая строка (`QAbstractItemView::SelectRows`).
*   **Редактор карточки (Нижняя половина, отделена через `QSplitter`):**
    *   Появляется только если выбрана строка в таблице.
    *   Разделен на две части (слева вопрос, справа ответ).
    *   Над каждым полем ввода небольшая панель форматирования (`QToolBar`): `[Жирный Ж]`, `[Курсив К]`, `[Подчеркнутый Ч]`.
    *   Поля ввода: `QTextEdit`. Они генерируют HTML.
    *   *Логика сохранения:* Подписка на сигнал `textChanged`. Использование `QTimer::singleShot` (например, 500 мс) для сохранения введенного текста в модель карточки, чтобы не писать в JSON при каждом нажатии клавиши (Debounce).

### 3.4. Режим обучения (Learn Mode)
*Открывается в `QStackedWidget` при нажатии на ▶️.*
*   **Логика выборки карточек:** При старте режима формируется очередь (основана на свойстве `NextReviewDate` <= Текущая дата).
*   **Верхняя панель:** 
    *   Кнопка `[Иконка ⬅️]` (Назад / Завершить).
    *   `QProgressBar` прогресса сессии.
*   **Центр (Отображение карточки):**
    *   Большой виджет (с тенью `QGraphicsDropShadowEffect`, имитирующий физическую карточку).
    *   Внутри `QTextBrowser` для безопасного отображения Rich Text (отцентрирован, крупный шрифт).
*   **Нижняя панель (Управление):**
    *   **Состояние 1 (Вопрос):** ОДНА большая кнопка по центру `[Иконка 👁️] Показать ответ` (Горячая клавиша - Пробел).
    *   **Состояние 2 (Ответ):** `QTextBrowser` обновляется, добавляя горизонтальную черту `<hr>` и текст ответа. 
    *   Появляются 4 кнопки с иконками и цветом (Реализация алгоритма):
        1.  **Снова (Красная, ✖️):** Забыл. Карточка уходит в конец очереди на сегодня. `Repetitions = 0`, `Interval = 1`.
        2.  **Трудно (Оранжевая, 😟):** `Interval` = текущий `Interval` * 1.2.
        3.  **Хорошо (Синяя, 🙂):** (Пробел). `Interval` = `Interval` * `EaseFactor`.
        4.  **Легко (Зеленая, ⭐):** `EaseFactor` += 0.15. `Interval` = `Interval` * `EaseFactor` * 1.3.
    *   *Под кнопками мелким шрифтом пишется вычисленный интервал (например: "1 д.", "4 д.", "15 д."). При нажатии карточка скрывается, появляется следующая.*
*   **Экран завершения:** Если очередь пуста, по центру выводится `[Иконка 🎉]` и текст "На сегодня всё!".

## 4. Алгоритм интервального повторения (Backend C++)
Реализуется в виде отдельного C++ класса `SpacedRepetitionLogic`.
*   При создании новой карточки: `Repetitions = 0`, `Interval = 1`, `EaseFactor = 2.5`.
*   Если пользователь нажимает "Снова":
    *   `Repetitions = 0`
    *   `Interval = 1`
    *   `EaseFactor = max(1.3, EaseFactor - 0.20)`
*   Если нажимает "Трудно / Хорошо / Легко":
    *   `Repetitions += 1`
    *   `Interval` вычисляется по формулам из п. 3.4.
    *   `NextReviewDate` = Текущая дата + `Interval` (в днях).

## 5. Требования к коду и безопасности (Ограничения)
1.  **Управление состояниями:** В приложении должен быть синглтон или корневой менеджер `ProjectManager`, который держит в памяти распарсенный JSON (как C++ структуры `std::vector<Deck>`).
2.  **Грязный флаг (isModified):** Любое изменение текста, добавление карточки, изменение интервала переводит флаг `isModified` в `true`.
3.  **Автосохранение:** Реализовать `QTimer`, который раз в 3 минуты проверяет флаг `isModified`. Если `true` — фоново (или быстро синхронно) перезаписывает `.flcr` файл.
4.  **Слоты и сигналы:** Строгое использование нового синтаксиса сигналов и слотов на указателях (Qt 5/6 syntax: `connect(sender, &Sender::signal, receiver, &Receiver::slot)`). Никаких макросов `SIGNAL()` и `SLOT()`.
5.  **Smart Pointers:** Данные внутри моделей должны управляться через `std::shared_ptr` или `std::unique_ptr` для предотвращения утечек памяти.
6.  **QAction и ярлыки:** Все основные действия (сохранение, добавление) реализуются через `QAction` для единой логики в меню и на тулбарах, с привязкой `QKeySequence` (горячих клавиш).
7. Соблюдение принципов ООП и MVC.
